# 旅行意图详细规则

当目的地决策需要细分类目、判断项、风险提示或示例时读取本文件。回答应始终服务于“去哪里更适合”的判断，不要漂移成完整日程攻略。

## 目录

- 现实条件规则
- 输出详细度规则
- 灵感触发规则
- 灵感关系强度规则
- 自然奇观规则
- 现场事件规则
- 活动体验规则
- 长期清单规则
- 随机组合规则
- 风险与不确定性规则

## 输出详细度规则

如果用户只是问可行性，先给结论，再给原因。

如果用户要完整推荐，使用完整分层格式。

如果用户要随机，只输出随机组合、推荐目的地、解释和可替换项。

如果用户正在连续迭代，继承上一轮标签，不重复完整说明。

如果信息不足，不要输出过长目的地清单。优先给 2-4 个方向，并说明每个方向还需要哪些条件才能判断。

## 现实条件规则

现实条件用于判断：能不能去、适不适合去、值不值得去、有什么必须避开。

条件包含时间、天数、预算、出发地、签证、到达交通、当地交通、驾照与租车、同行人、舒适度和避开项。

### 时间规则

时间会影响气温、降水、台风、沙尘暴、暴雪、酷暑、人流、价格、花期、动物迁徙、奇观可见性、事件是否发生、景区是否开放。

必须判断用户选择的月份是否适合、是否是旺季、是否有天气风险、是否和奇观/事件窗口匹配、是否有更好的月份。

### 天数规则

天数决定目的地半径：

```
1-2 天：城市周边、短途自然点
3-4 天：邻近城市、短途国际
5-7 天：一个区域或国家经典路线
8-10 天：多城市、小环线
11-14 天：深度旅行、跨区域路线
15 天以上：长线、国家公园、洲际路线
```

必须判断这个目的地用这个天数值不值得、是否路上时间过长、是否容错不足、是否不如换近一点的目的地。

### 预算规则

预算不能只看当地物价。必须考虑往返交通、住宿、餐饮、当地交通、门票、活动、签证、租车、保险、装备、旺季溢价和隐形成本。

例如：

- 泰国当地便宜，但机票可能影响总预算。
- 冰岛景点很多免费，但住宿、租车、餐饮贵。
- 美国国家公园门票不贵，但租车、油费、住宿会拉高成本。

### 出发地规则

同一个目的地，对不同出发地完全不同。需要考虑飞行时间、直飞可能、转机次数、机票成本、时差、短假是否值得、签证便利度、是否能火车/高铁到达。

如果用户没有提供出发地，可以先基于一般情况回答，并说明补充出发地会更准。

### 签证规则

签证影响说走就走程度、临时出发可行性、多国路线可行性、材料准备难度、办理周期和过境签风险。

签证偏好可包括只看免签、接受落地签、接受电子签、可以办普通签证、不想准备复杂材料、可以接受长办理周期、已有某些签证。

### 交通规则

交通分为到达交通和当地交通。

到达交通包括直飞优先、可接受一次转机、可接受多次中转、只看火车/高铁、可接受飞机加船、可接受小飞机、不想红眼航班、不想长途飞行。

当地交通包括公共交通必须方便、不想自驾、可以自驾、可以打车、可以包车、可以跟团、可以坐船、想公路旅行、想火车旅行、想邮轮/跳岛、不想频繁换交通、不想每天搬行李、不想长时间坐车。

不自驾应该是强条件。如果目的地强依赖自驾，要降权或放入“不太建议”。

### 驾照与租车规则

必须区分四件事：

```
用户想不想开车
法律上能不能开车
能不能租到车
目的地是否适合自驾
```

判断内容包括驾照签发地、是否有国际驾照/国际驾驶许可、是否有官方翻译件、是否有信用卡、年龄、最低租车年龄、左行/右行、道路难度、停车难度、雪路/山路/夜路风险、事故处理难度、不自驾替代方案。

不要因为用户说“可以自驾”就默认目的地适合。

### 同行人规则

同行人影响目的地适配：

```
一个人：安全、交通、住宿、社交氛围
情侣：氛围、住宿、拍照、节奏
朋友：活动、夜生活、预算弹性
带小孩：卫生、安全、亲子设施、步行强度
带父母：少换酒店、少走路、医疗、饮食适应
宠物：入境、住宿、交通、景点限制
```

### 舒适度规则

舒适度不同于预算。用户可能预算高但能吃苦，也可能预算低但不想折腾。

舒适度包括住宿舒适、饮食适应、不频繁换酒店、不走太多路、不坐太久车、医疗方便、网络稳定、语言沟通简单、能接受露营、能接受高海拔、能接受极寒/极热。

### 避开项规则

避开项通常应该强过滤。包括太热、太冷、高湿度、雨季、台风季、沙尘暴、暴雪、治安差、地震高风险、火山风险、高海拔、人太多、过度商业化、语言障碍、饮食不适应、每天换酒店、必须自驾、长途飞行、红眼航班、隐形消费多。

## 灵感触发规则

灵感用于回答“什么让我想出发”。灵感不是简单兴趣匹配，而是要判断发源地、代表产区、真实取景地、作品原型地、氛围相似地、人物相关地、主题活动地、是否值得专程、是否更适合顺路体验。

## 灵感关系强度规则

判断灵感与目的地的关系强度：

- 强关联：发源地、真实取景地、主要发生地、官方或广泛认可的核心地点。
- 中关联：代表性城市、重要产区、人物长期生活/创作地、氛围高度相似。
- 弱关联：仅有相似氛围、商业包装、普通游客感知不强。

推荐时要说明：

- 是否值得专程。
- 是否更适合顺路。
- 是否只适合爱好者。
- 普通游客是否也会觉得有价值。

例如，新加坡司令和新加坡/莱佛士酒店是强关联，但更常作为城市旅行中的顺路体验；青春猪头少年和镰仓/藤泽/江之岛的作品场景关联更强，对爱好者的专程价值明显更高。

### 酒

包括经典鸡尾酒、酒类产区、酒吧文化、酒店酒吧、蒸馏厂、酒庄、精酿啤酒城市、清酒酒藏。

判断示例：

- 新加坡司令 -> 新加坡，强关联，但可能更适合城市顺路体验。
- 自由古巴 -> 古巴/哈瓦那，文化氛围强。
- 威士忌 -> 苏格兰、爱尔兰、日本山崎、肯塔基等。

必须判断是否值得专程。

### 食物

包括披萨、寿司、拉面、越南粉、咖喱、提拉米苏、牛角包、汉堡、火锅、海鲜、巧克力、咖啡、茶、奶酪、甜品、夜市、米其林、街头小吃。

判断发源地、代表城市、美食密度、游客友好度、是否预约、饮食适应、是否适合父母/孩子、是否适合作为主旅行理由。

### 动漫

包括具体作品、真实取景地、圣地巡礼、氛围相似地、主题展览、周边购物、动漫街区。

判断作品关联强度、是否打扰居民、是否适合普通游客、交通是否方便、是否适合专程、是否适合顺路。

例如：

- 青春猪头少年 -> 镰仓/藤泽/江之岛，强关联。
- 灌篮高手 -> 镰仓高校前等地，强取景地属性。

### 电影和电视剧

判断真实取景地、故事发生地、电影氛围、导演/演员相关地、电影节/展览、是否适合普通游客。

例如：

- 疑犯追踪 -> 纽约，偏都市犯罪剧氛围、街区、地铁、夜景、城市监控感。
- 哈利波特 -> 英国多地、伦敦、牛津、爱丁堡等。

### 游戏

判断现实原型、相似地貌、相似城市氛围、游戏展、电竞赛事、游戏公司/博物馆、主题咖啡。

例如：

- 刺客信条 -> 历史城市、古城、地中海等。
- 如龙 -> 日本城市街区氛围。
- 荒野大镖客 -> 美国西部荒野/国家公园氛围。

### 小说、音乐、建筑、人物、博物馆、生活方式

都要判断关联类型、关联强度、是否值得专程、适合顺路还是主题旅行、普通游客是否也能享受、与用户条件是否匹配。

不要因为“喜欢某作家”就直接推荐一个城市，而要解释关系。

## 自然奇观规则

奇观用于回答“我想亲眼看见什么”。奇观是自然、天文、生态、地貌、季节或气象现象。

### 奇观分类

```
天空/天文：极光、日全食、日环食、月食、银河、流星雨、彗星、行星合月、极昼、极夜
海洋：荧光海、鲸鱼迁徙、海龟产卵、沙丁鱼风暴、潮汐路、蓝洞、珊瑚礁
地质：火山、冰洞、间歇泉、冰川、峡谷、盐湖、沙漠、丹霞、喀斯特、温泉地热
季节：樱花、红叶、薰衣草、郁金香、雪墙、沙漠花海、蓝花楹、紫藤、银杏、雪景
生物：萤火虫、企鹅、蝴蝶迁徙、野生动物大迁徙、雪猴、北极熊、鲑鱼洄游、观鸟
气象：云海、雾凇、雷暴、沙尘暴、龙卷风观测、极寒景观、彩虹、幻日
```

### 奇观与事件边界

日全食、日环食、月食、流星雨峰值、彗星最佳观测、行星合月属于奇观。

只有人为组织的活动才属于事件，例如日食观测节、天文台观测活动、天文营、流星雨观测活动。

### 奇观判断项

必须考虑最佳时间、可见区域、成功率、天气依赖、光污染、月相、海况、潮汐、季节波动、停留天数、安全风险、是否需要向导、是否适合专程、失败时是否有替代玩法、出发前应查看什么预报。

### 奇观输出规则

不要说“你一定能看到极光”“这里一定能看到鲸鱼”“这个时间一定有樱花”。应该说“更有机会”“通常更适合”“成功率较高”“受天气影响明显”“建议停留几晚增加机会”“需要出发前查看预报”。

## 现场事件规则

事件用于回答“我想赶上什么现场”。事件必须是人为组织的现场活动。

### 事件分类

```
航空航天：航展、火箭发射、航天开放日、飞行表演、航空博物馆活动
体育赛事：F1、世界杯、奥运会、NBA、MLB、NFL、网球公开赛、马拉松
音乐演出：演唱会、音乐节、巡演、爵士节、歌剧季、小型现场演出
节庆庆典：啤酒节、雪祭、花火大会、圣诞集市、狂欢节、新年倒计时
展览展会：车展、游戏展、动漫展、艺术双年展、设计周、书展、科技展
文化活动：电影节、文学节、戏剧节、宗教庆典、博物馆之夜、文化遗产活动
```

### 事件判断项

必须考虑日期、地点、是否对公众开放、门票难度、预约、票价、住宿涨价、交通管制、人流、安全、语言门槛、取消风险、延期风险、如果活动失败城市是否仍然值得去。

### 火箭发射特殊规则

火箭发射必须提示发射可能延期或取消、需要查看官方发射窗口、最好预留备用天数、观测点/停车/封路需要提前确认、如果发射失败周边是否有航天中心/博物馆/海滩等替代体验。

### 航展特殊规则

航展必须考虑公众开放日、专业日限制、飞行表演是否每天都有、天气影响、交通管制、住宿压力、摄影点、亲子友好度。

## 活动体验规则

活动用于回答“我想在那里做什么”。

### 活动分类

```
雪地：滑雪、单板、雪地摩托、雪鞋徒步、狗拉雪橇、冰洞、雪国列车
水下：潜水、浮潜、自由潜、沉船、珊瑚礁、蓝洞、鲸鲨、蝠鲼
海上：冲浪、帆船、皮划艇、邮轮、跳岛、海钓、观鲸船、海岸公路
山地：徒步、登山、攀岩、峡谷、高山湖泊、雪山观景、缆车、朝圣古道
荒野：露营、房车、国家公园自驾、野生动物、沙漠、雨林、极地、星空营地
城市：美食、购物、夜生活、博物馆、建筑漫游、城市散步、书店、市集
放松：温泉、海岛躺平、度假村、水疗、森林疗愈、湖边、乡村慢游
亲子：主题乐园、动物园、水族馆、科学馆、轻户外、铁路体验、亲子度假村
摄影：星空、雪山、城市夜景、街头、野生动物、建筑、花季、极光、沙漠
```

### 活动强度规则

必须区分活动类型和强度：

```
1. 轻松体验
2. 普通运动
3. 中等挑战
4. 高强度挑战
5. 专业门槛
```

滑雪可以是新手课，也可以是野雪。徒步可以是短步道，也可以是多日高海拔。潜水可以是浮潜，也可以是洞潜。不要默认用户要高强度。

### 活动判断项

必须考虑季节、强度、经验、证书、装备、教练、向导、安全、天气、海况/雪况/路况、交通、是否不自驾可行、是否适合亲子/父母、是否值得专程。

## 长期清单规则

清单用于回答“我想完成什么长期旅行目标”。不要只列完整清单，要推荐下一站、当前最适合完成哪一个、是否适合一次打卡多个、按季节怎么安排、按交通效率怎么安排、按难度怎么安排、按代表性怎么安排。

### 清单分类

```
自然清单：国家公园、世界自然遗产、火山、峡谷、冰川、沙漠、湖泊
文化清单：世界文化遗产、古城、宗教建筑、城堡、历史遗址
娱乐清单：迪士尼、环球影城、乐高乐园、主题乐园、影视乐园
体育清单：F1 分站、网球公开赛、足球主场、奥运场馆、马拉松
航天清单：火箭发射场、航天博物馆、天文台、太空中心
交通清单：著名铁路、公路路线、观景列车、邮轮航线、渡轮、缆车
作品清单：动漫圣地、电影取景地、电视剧取景地、游戏原型地、文学城市
美食与酒清单：鸡尾酒发源地、葡萄酒产区、威士忌产区、咖啡城市
```

例如用户说“我想游览所有国家公园”，需要进一步判断范围是哪个国家、从哪里出发、几天、是否自驾、想看什么地貌、是否带父母/小孩、想先去容易的还是代表性的。

如果信息不足，可以先给策略，而不是硬列一堆。

## 随机组合规则

随机用于回答“我不知道想去哪，你帮我组合一个旅行理由”。

随机的不是城市，而是旅行理由组合。

例如：

```
海边电车 + 动漫氛围 + 轻松散步 -> 镰仓 / 江之岛
极光 + 温泉 + 不自驾 + 冬季 -> 特罗姆瑟 / 芬兰拉普兰
威士忌 + 海岛 + 小镇 + 秋天 -> 苏格兰艾雷岛
鸡尾酒 + 城市夜景 + 美食 + 短途 -> 新加坡
```

随机模式包括完全随机、带限制随机、按主题随机、旅行盲盒、锁定部分条件后随机、保留目的地换玩法、保留灵感换目的地。

随机组合必须现实可成立、旅行上有意义、能解释原因、不违反用户锁定条件、不包装高风险体验。

不要生成明显不可行组合，例如“极光 + 热带海岛 + 2 天周末 + 不坐飞机”。

## 风险与不确定性规则

必须明确天气风险、台风、沙尘暴、暴雪、酷暑、高海拔、火山、地震、野火、洪水、泥石流、治安、医疗不便、交通中断、活动延期、门票售罄、花期波动、动物不可控、极光不可控、火箭发射延期。

对于高风险或低确定性内容，应给出：

```
不建议只为它专程
建议增加备用天数
建议安排替代玩法
建议出发前查官方信息
```
